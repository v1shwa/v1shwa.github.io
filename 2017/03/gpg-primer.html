GPG Primer

15 Mar 2017

GPG (GNU Privacy Guard) is a free and open-source implementation of PGP. GPG can be used to encrypt and the sign the sensitive data such as password, emails and files. In this post, we will discuss the quickest way to install, create, export, import, encrypt & decrypt files & text data using GPG.

How GPG works:

  • GPG is an implementation of public key cryptography â€” which has a pair of keys called public key and private key.
  • Public key is shared with others which can be used only to encrypt the data.
  • Private key is not shared & is known* *only by owner. Private key is used to decrypt the file that is encrypted using its public key.

Installing GPG:

  • Windows: https://www.gpg4win.org/download.html
  • Mac: brew install gnupg
  • RHEL/CentOS: yum install gnupg
  • Debian/Ubuntu: apt-get install gnupg

Creating your GPG key pair:

  • To generate a new key pair, use the following command. This generates a new key pair (public & private) key and stores the data a keyring.

      gpg --gen-key

  • For better security, when prompted

    • choose RSA key
    • set the key size to 4096
    • Choose a very long & non-obvious password

Exporting the public key:

  • To save your public key to a file, run:

      gpg --export --armor you@example.com > mykey.asc
  • This exports your public (not your secret private key) to mykey.asc file on your computer. This key can be shared safely with people who wants to send you secret data.

  • ðŸ’¡ This public key can also be uploaded to common key server using

      gpg --send-keys PUB-KEY-ID

Importing public key:

  • If you have a public key received from someone, you can use it to encrypt the data on your computer & send it to them. To do so, you need to import the key locally to your machine.

  • To import a key file called mykey.asc, run:

      gpg --import --armor mykey.asc

  • ðŸ’¡ If you have received a public key-id instead of key itself, you can import it from the server using

      gpg --recv-keys PUB-KEY-ID

Encrypting a document:

  • To encrypt a document, run:

      gpg --recipient sender_email@example.com --encrypt file_to_encrypt.doc
  • This will create a file named file_to_encrypt.doc.gpg ,which you can share it with the recipient.
  • ðŸ’¡ You can use GPG to encrypt files that can be decrypted only by you.

Decrypting a document:

  • If you have a file (say, myfile.doc.gpg) that was encrypted using your public key, you can decrypt it using:

      gpg --decrypt myfile.doc.gpg > myfile.doc

  • The decrypted file will be saved as myfile.doc on your computer.

Other Useful commands:

  • To list all public keys on your computer : gpg --list-keys
  • To list all secret keys on your computer : gpg --list-secret-keys
  • To delete a public key from your computer : gpg --delete-key email@example.com
  • To delete a secret key from your computer : gpg --delete-secret-key you@example.com
